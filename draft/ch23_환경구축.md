# 23장. 개발 환경 구축 — PyTorch · CUDA · CARLA · ROS2

> **학습 목표**

## 23.1 어떤 하드웨어를 준비해야 하는가

이 책의 실습은 넓은 범위를 다룬다. 1부의 개념 학습은 GPU 없이도 따라갈 수 있지만, 2부 이후의 실습부터는 GPU가 필요하다. 가장 현실적인 기준을 제시하자면, 2장부터 8장까지의 비전 기초 실습은 RTX 3060 12GB나 Colab Pro로 충분하다. 9장부터 17장까지의 예측·분산 실습은 RTX 4080 급이 편안하며, 이상적으로는 H100 1장이 있으면 좋다. 18~22장의 Physical AI 주제는 RTX 4090급의 메모리가 권장되고, 27장의 Isaac Lab은 더 강력한 GPU가 필요한 경우가 있다.

Colab Pro는 월 1~2만 원대로 1부와 2부의 실습을 대부분 커버한다. 한계는 24시간 연속 세션이 끊긴다는 점과, 대형 학습에는 시간이 부족하다는 점이다. 본격적인 실습자는 중고 RTX 3060이나 새 RTX 4070 정도를 마련하는 것이 장기적으로 유리하다. 이 이야기의 교훈은, **하드웨어는 책을 읽으면서 점진적으로 갖추면 된다**는 것이다. 처음부터 모든 것을 갖출 필요는 없다.

## 23.2 Windows + WSL2 — 한국 독자에게 가장 친숙한 경로

한국 독자의 대부분은 Windows 노트북이나 데스크탑을 사용한다. 이 환경에서 가장 안정적인 개발 환경이 **WSL2(Windows Subsystem for Linux 2)** 다. Linux의 유연성과 Windows의 친숙함을 동시에 얻을 수 있다.

설치 절차는 몇 년 전보다 훨씬 단순해졌다. PowerShell을 관리자 권한으로 열고 `wsl --install -d Ubuntu-22.04`를 실행하면, 수 분 안에 Ubuntu 22.04가 설치된다. 그 뒤 `wsl --update`로 최신 커널로 업데이트한다. NVIDIA GPU를 쓰려면 **Windows 측에 Studio Driver**를 설치해야 한다. WSL2는 Windows의 드라이버를 그대로 공유하므로 Linux 내에 별도 GPU 드라이버를 설치하지 않는다. 이 구조가 처음 접하는 이에게 혼란을 주는 지점이니 기억해 두자.

WSL2 내부에 들어가면 Linux 환경이다. CUDA Toolkit을 설치하려면 NVIDIA의 공식 저장소에서 WSL 전용 deb 패키지를 다운로드해 설치한다.

```bash
wget https://developer.download.nvidia.com/compute/cuda/repos/wsl-ubuntu/x86_64/cuda-keyring_1.1-1_all.deb
sudo dpkg -i cuda-keyring_1.1-1_all.deb
sudo apt update
sudo apt install -y cuda-toolkit-12-4
```

Python 환경은 uv 또는 conda 둘 다 좋다. 저자는 2024년부터 uv를 기본으로 쓴다. 가볍고 빠르기 때문이다.

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
uv venv --python 3.11
source .venv/bin/activate
uv pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124
```

설치 검증은 Python에서 `import torch; print(torch.cuda.is_available())`를 실행해 True가 나오면 끝이다. 첫 성공의 순간이 전체 학습의 출발점이다.

## 23.3 Ubuntu 네이티브 — CARLA와 Isaac Lab을 위한 선택

WSL2는 편리하지만, CARLA 시뮬레이터와 Isaac Lab 같은 무거운 그래픽 애플리케이션에서는 불안정할 수 있다. GPU 가상화의 한계 때문이다. 이 무거운 실습을 본격적으로 할 생각이라면 **Ubuntu 22.04를 네이티브**로 설치하는 편이 안정적이다.

듀얼 부트 설정은 Windows 설치 후 파티션을 나누어 Ubuntu를 추가 설치하는 방식이 가장 일반적이다. Ubuntu 22.04 기준으로 NVIDIA 드라이버 설치는 간단하다. `sudo ubuntu-drivers autoinstall`로 자동 설치한 뒤 재부팅하면 준비가 된다. 이후의 CUDA와 PyTorch 설치는 WSL2와 동일하다.

네이티브 Ubuntu의 장점은 안정성 외에도 있다. 시스템 리소스를 Windows와 나누지 않으므로 GPU와 메모리를 최대한 활용할 수 있고, 서버처럼 24시간 학습을 돌려 놓기에도 편하다. 저자의 개발용 데스크탑은 Ubuntu 네이티브이고, 문서 작업용 노트북은 Windows + WSL2 조합이다. 이 구분이 각자의 장점을 취하는 방식이다.

## 23.4 Colab과 클라우드 — 하드웨어 없이 시작하기

하드웨어 구매가 부담스럽다면 Colab Pro가 가장 빠른 출발점이다. 구글 계정만 있으면 무료 버전도 쓸 수 있지만, T4 GPU의 세션 제한 때문에 실제 학습에는 부족하다. Pro(월 약 14,000원)로 업그레이드하면 L4나 A100 같은 더 나은 GPU에 접근할 수 있다. 1~8장의 실습 대부분이 Colab에서 문제없이 돌아간다.

Colab의 장점은 **초기 비용이 거의 없다**는 것과, **환경이 이미 세팅되어 있다**는 것이다. PyTorch, CUDA가 기본으로 설치되어 있어 `import torch`만 하면 바로 된다. 단점은 세션이 끊기면 메모리가 날아가고, 24시간 연속 학습이 불가능하다는 것이다. 본격 학습에는 로컬 GPU나 AWS·GCP의 장기 인스턴스가 더 맞다.

대학원생이나 연구원이라면 학교나 기관의 GPU 클러스터를 먼저 알아볼 것을 권한다. 무료로 쓸 수 있는 자원이 의외로 많다.

## 23.5 CARLA 설치 — 14장 이후 시뮬레이션의 필수 도구

CARLA는 자율주행 시뮬레이션의 사실상 표준이다. 0.9.15 버전이 2024~2025년 기준 안정적이다. 설치는 간단하다.

```bash
mkdir -p ~/carla && cd ~/carla
wget https://carla-releases.s3.us-east-005.backblazeb2.com/Linux/CARLA_0.9.15.tar.gz
tar -xvzf CARLA_0.9.15.tar.gz
./CarlaUE4.sh -quality-level=Low -prefernvidia
```

CARLA 서버가 실행되면, Python 클라이언트로 연결한다. `pip install carla==0.9.15`로 Python 라이브러리를 설치하고, 간단한 스크립트로 연결 확인을 한다.

```python
import carla
client = carla.Client('localhost', 2000)
world = client.get_world()
print(world.get_map().name)
```

맵 이름이 출력되면 연결 성공이다. 이후 14장과 24~26장의 실습에서 CARLA를 다양한 시나리오로 활용한다. 주의할 점은 CARLA가 Unreal Engine 기반이라 GPU 메모리를 많이 쓴다는 것이다. 최소 8GB VRAM이 필요하고, 쾌적한 사용을 위해서는 12GB 이상을 권장한다.

## 23.6 ROS2 Humble — 자율주행 업계의 공용어

30장에서 상세히 다루지만, ROS2 Humble의 설치를 여기서 먼저 짚어둔다. Ubuntu 22.04가 ROS2 Humble의 공식 지원 OS다. 설치는 여러 단계가 있지만, 명령만 따라가면 20분 안에 끝난다.

```bash
sudo apt install -y software-properties-common
sudo add-apt-repository universe
sudo apt update && sudo apt install -y curl gnupg lsb-release
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
  -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
  http://packages.ros.org/ros2/ubuntu $(lsb_release -cs) main" \
  | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
sudo apt update
sudo apt install -y ros-humble-desktop
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
```

설치 후 새 터미널에서 `ros2 --help`가 동작하면 성공이다. ROS2의 각 패키지는 apt로 설치할 수 있다(예: `sudo apt install ros-humble-nav2-bringup`). 30장에서 MentorPi와 CARLA를 ROS2로 엮는 실습을 본격적으로 다룬다.

## 23.7 Isaac Lab — 27장 실습을 위한 준비

Isaac Lab은 NVIDIA Omniverse 기반의 무거운 시뮬레이터다. 설치가 복잡하고 하드웨어 요구도 크다. RTX 3060 이상, 가급적 4070 이상을 권장한다. 공식 가이드를 따르는 것이 가장 안전하며, 여기서는 개요만 짚는다.

```bash
git clone https://github.com/isaac-sim/IsaacLab.git
cd IsaacLab
./isaaclab.sh --install
./isaaclab.sh --python scripts/environments/list_envs.py
```

`--install` 스크립트가 Omniverse Kit과 필요한 파이썬 패키지를 자동 설치한다. 수 기가바이트의 다운로드가 필요하므로, 네트워크가 충분히 빨라야 한다. 설치가 끝나면 Go2, Franka, H1 같은 사전 정의된 환경들이 목록에 표시된다. 27장에서 이들을 실제로 학습시키고 시각화한다.

## 23.8 Docker로 재현성 확보하기

연구·협업·시험 제출 같은 맥락에서 **환경의 재현성**이 중요해진다. 내 노트북에서 돌던 코드가 리뷰어의 노트북에서는 돌지 않는 상황을 피하려면 Docker가 가장 안정적인 해법이다.

```dockerfile
FROM nvidia/cuda:12.4.1-cudnn-devel-ubuntu22.04

RUN apt-get update && apt-get install -y \
    python3.11 python3.11-venv git build-essential \
    libgl1 libglib2.0-0 ffmpeg

WORKDIR /workspace
COPY requirements.txt .
RUN python3.11 -m venv .venv && \
    . .venv/bin/activate && \
    pip install -r requirements.txt

ENV PATH="/workspace/.venv/bin:$PATH"
CMD ["bash"]
```

빌드와 실행은 표준적이다.

```bash
docker build -t tesla-book:latest .
docker run --gpus all -it -v $(pwd):/workspace tesla-book:latest
```

이 이미지는 저자 GitHub 저장소에서도 제공될 예정이다. 독자가 직접 빌드하지 않고 `docker pull leelang7/tesla-book` 명령 한 번으로 가져다 쓸 수 있게 한다.
